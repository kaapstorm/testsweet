# Design: file-path argument + multi-target dispatch + selectors

## Scope

Extend the `python -m assertions` CLI to accept one or more targets,
each of which can be a file path, a dotted module, or a dotted
selector that picks a subset of tests within a module. Add a
selectors-aware mode to `run(module, names=None)`.

In scope:

- File-path argument (`tests/foo.py`, `./tests/foo.py`, `/abs/foo.py`).
- One or more targets on the command line.
- Selectors: `module.func`, `module.Class`, `module.Class.method`.
- Multi-selector dispatch: same module with different selectors
  combined into a single `run` call.
- Unmatched selectors raise `LookupError`; the CLI lets it propagate
  as a regular Python traceback, matching the existing handling of
  `ModuleNotFoundError`.

Out of scope for this slice:

- Bare `python -m assertions` (no args) — still exits 2 with usage.
- Directory arguments (`python -m assertions tests/`).
- Walking the filesystem for test files.
- Parameterized-tuple selectors (`module.func[0]`).
- Globs, wildcards, regex patterns.
- Pretty-printing `LookupError` / `ModuleNotFoundError` errors —
  deferred to a later cleanup slice.

## Approach

A new module `_targets.py` parses each argv entry into a
`(module, names)` pair where `names` is `None` (run all) or a list of
selector tails (e.g., `['foo']`, `['MyClass.method']`). The CLI walks
argv, collects pairs, groups by module identity (preserving argv
order of first appearance), and calls `run(module, names=...)` once
per group.

`run` gains a `names` parameter. Inside `run`, the discovery loop
matches each unit against the supplied names. Names that match are
included; names not matched by any unit cause `LookupError` after the
run finishes (i.e., results are returned for matched units before the
error fires? — see "Error timing" below).

## Public surface

```python
from assertions import run


run(module)                       # all tests (current behavior)
run(module, names=['foo'])        # one free function
run(module, names=['MyClass'])    # all methods of a class
run(module, names=['Class.x'])    # one method
run(module, names=['Class.x',
                   'Class.y'])    # two methods
run(module, names=['foo'])        # raises LookupError if no `foo`
```

CLI:

```
python -m assertions <target> [<target>...]
```

## Behavior

### Target classification

Each argv entry is classified by these rules, in order:

1. Contains `'/'` or ends with `'.py'` → **file path**.
2. Otherwise → **dotted target**: importable as a module, or a
   selector with 1 or 2 trailing segments.

### File path

- Resolve via `pathlib.Path(target).resolve()` (handles `.`, `..`,
  relative paths).
- Use `importlib.util.spec_from_file_location` with the path's stem
  as the module name.
- Run `run(module)` with no `names`.

If the file does not exist, `spec_from_file_location` returns `None`
or import fails — the resulting exception (`FileNotFoundError`,
`ImportError`) propagates.

### Dotted target

Walk from the longest prefix to the shortest, attempting
`importlib.import_module`:

1. Try the whole string. If it imports → `(module, None)`.
2. Else strip the last segment. If the prefix imports → that's the
   module; the stripped segments are the selector tail.
3. If two segments were stripped (`module.Class.method`) →
   `(module, ['Class.method'])`.
4. Three or more segments stripped → `LookupError(f'cannot resolve
   {target!r}: too many trailing segments')`.
5. No prefix imports → the original `ModuleNotFoundError` from the
   first attempt propagates.

### Multi-target grouping

The CLI processes argv in order:

1. Resolve each target into `(module, names)` (where `names` is
   either `None` or a list of selector tails).
2. Group by module identity, preserving the order in which each
   module first appeared.
3. Within a group, merge `names`:
   - If any entry has `names=None`: drop all the others; this group
     runs unfiltered.
   - Otherwise: concatenate the lists.
4. For each group, call `run(module, names=merged_names)`.
5. Collect and print results in group order; lines for each group
   appear in the order produced by that `run` call.

### `run(module, names=None)`

```python
def run(
    module: ModuleType,
    names: list[str] | None = None,
) -> list[tuple[str, Exception | None]]:
    ...
```

Behavior when `names is None`: identical to today.

Behavior when `names` is a list:

1. Call `discover(module)` to get the units.
2. Build a per-class method-filter from `names`. For each entry:
   - If it has no dot → it's a "whole unit" name (function or class).
   - If it has one dot → it's a `Class.method` selector.
3. Validate every name against the discovered units:
   - A whole-unit name matches if some unit's `__qualname__` equals
     it.
   - A `Class.method` name matches if some discovered unit is a class
     with the given name AND that class has a method by that name in
     `_public_methods(unit)`.
   - Names that don't match any unit go into `unmatched`.
   - If `Class` is referenced both as whole-unit and as
     `Class.method`, the `Class.method` selector counts as matched
     (it's redundant, not unmatched).
4. If `unmatched` is non-empty: raise
   `LookupError(f'no test units matched: {sorted(unmatched)}')`.
   Nothing has been run yet.
5. Iterate units in discover order. For each unit:
   - If the unit's `__qualname__` is referenced as a whole-unit name:
     run it as today (all methods of a class, all params of a
     parameterized function).
   - Else if the unit is a class referenced by one or more
     `Class.method` selectors: run only those methods, in `vars()`
     definition order.
   - Else: skip.

Conflict rule (whole unit + method forms for the same class):

If `names=['MyClass', 'MyClass.x']`, the class form wins — all methods
of `MyClass` run. The `MyClass.x` selector is treated as redundant
(matched, not unmatched).

### Error timing

`LookupError` for unmatched names is raised BEFORE any tests run.
`run` first calls `discover(module)`, validates that every name in
`names` matches at least one discovered unit (or, for `Class.method`,
that the class is discovered and the method exists on it), and only
then executes. If validation fails, no tests are run and no results
are produced — `run` raises. This avoids the alternative ("run what
matched, error at the end") which would either lose the matched
output (`run` can't both return and raise) or require a stateful
side-channel.

A consequence: if a target combines matched and unmatched names, the
user sees the error and zero matched output; they'll have to fix the
typo and re-run.

## Module layout

| File | Action | Purpose |
|------|--------|---------|
| `src/assertions/_targets.py` | create | `parse_target(s)`; file vs dotted resolution |
| `src/assertions/_runner.py` | modify | `run(module, names=None)` + filter + LookupError |
| `src/assertions/__main__.py` | modify | Multi-arg CLI; group by module |

`_targets.py` owns target parsing and module loading by path.
Keeping it separate from `__main__.py` makes the parsing logic
unit-testable without going through `subprocess`.

## Tests

### `tests/test_targets.py` (new)

1. `parse_target('tests.fixtures.runner.all_pass')` → `(module, None)`.
2. `parse_target('tests/fixtures/runner/all_pass.py')` → `(module, None)`
   — verifies file-path branch.
3. `parse_target('./tests/fixtures/runner/all_pass.py')` → works.
4. `parse_target('/abs/path/all_pass.py')` (constructed from
   `pathlib.Path(...).resolve()` of an existing fixture) → works.
5. `parse_target('tests.fixtures.runner.all_pass.passes_one')` →
   `(all_pass module, ['passes_one'])`.
6. `parse_target('tests.fixtures.runner.class_simple.Simple')` →
   `(class_simple module, ['Simple'])`.
7. `parse_target('tests.fixtures.runner.class_simple.Simple.first')`
   → `(class_simple module, ['Simple.first'])`.
8. `parse_target('tests.fixtures.runner.class_simple.Simple.first.extra')`
   → raises `LookupError`.
9. `parse_target('totally.not.a.module')` → raises
   `ModuleNotFoundError`.

### `tests/test_runner.py` (modify, additions)

Add `class TestRunNames(unittest.TestCase)`:

10. `run(mod, names=['passes_one'])` on `all_pass` → results contain
    only `passes_one`, not `passes_two`.
11. `run(mod, names=['Simple'])` on `class_simple` → all methods of
    `Simple` run (`Simple.first`, `Simple.second`).
12. `run(mod, names=['Simple.first'])` → only `Simple.first` runs.
13. `run(mod, names=['Simple.first', 'Simple.second'])` → both run,
    in vars order.
14. `run(mod, names=['Simple', 'Simple.first'])` → all methods of
    `Simple` run (class form wins).
15. `run(mod, names=['nonexistent'])` → raises `LookupError` whose
    message contains `'nonexistent'`.
16. `run(mod, names=['passes_one', 'nope'])` → raises `LookupError`
    listing `'nope'`. The matched name (`passes_one`) is NOT run
    (validation happens before execution); verify by patching the
    matched function with a tracker and confirming it was not called.
17. `run(mod, names=['adds'])` on `params_simple` → both
    `adds[0]` and `adds[1]` run (selector matches the function;
    params expand as today).
18. `run(mod, names=['Simple.nonexistent'])` on `class_simple` →
    raises `LookupError` (the class exists but the method doesn't).

### `tests/test_cli.py` (modify, additions)

19. File-path argv:
    `python -m assertions tests/fixtures/runner/all_pass.py` exits 0
    with `passes_one ... ok` and `passes_two ... ok` on stdout.
20. Selector argv:
    `python -m assertions tests.fixtures.runner.class_simple.Simple.first`
    exits 0 with `Simple.first ... ok` and NO `Simple.second` line.
21. Multiple module targets:
    `python -m assertions tests.fixtures.runner.all_pass
    tests.fixtures.runner.has_failure` exits 1, stdout contains lines
    for both modules.
22. Multiple selectors against same module:
    `python -m assertions tests.fixtures.runner.class_simple.Simple.first
    tests.fixtures.runner.class_simple.Simple.second` exits 0,
    stdout contains both lines, single grouped invocation
    (verifiable by counting that no method runs twice).
23. Unmatched selector:
    `python -m assertions tests.fixtures.runner.all_pass.nonexistent`
    exits non-zero; stderr contains `LookupError`.
24. Same module mixed unfiltered + selector:
    `python -m assertions tests.fixtures.runner.all_pass
    tests.fixtures.runner.all_pass.passes_one` runs the unfiltered
    form (both `passes_one` and `passes_two`), not just one.

## Acceptance

- `uv run python -m assertions tests/fixtures/runner/all_pass.py`
  exits 0 and prints `passes_one ... ok` / `passes_two ... ok`.
- `uv run python -m assertions
  tests.fixtures.runner.class_simple.Simple.first` runs only
  `Simple.first`; exit 0.
- `uv run python -m assertions tests.fixtures.runner.all_pass
  tests.fixtures.runner.has_failure` exits 1; lines for both modules.
- All previous tests still pass.
- `uv run mypy` reports no issues.
