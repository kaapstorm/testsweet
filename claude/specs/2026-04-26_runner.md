# Design: minimal runner and CLI

## Scope

A function that runs the `@test`-decorated callables in a module and a
thin command-line entry point that drives it.

Out of scope for this slice:

- Discovery from file paths or via package walking
- Bare `python -m assertions` (no argument) auto-discovery
- Single-test selectors (`tests.foo.bar:test_x`)
- `Test` base class support
- Parameterized tests (`test_params`)
- `assert_raises` / `assert_warns`
- Trailing summary lines (e.g. "3 passed, 1 failed")
- Distinguishing test failure from test error
- Capturing or formatting tracebacks for failed tests

## Approach

`run(module)` calls `discover(module)`, invokes each test with no
arguments, and records the outcome. The CLI imports the named module,
calls `run`, prints one line per result, and chooses an exit code.

The runner is deliberately small: catch `Exception`, let everything else
propagate. No printing, no logging. The CLI is the only thing that
formats output.

## Public surface

```python
from assertions import run

def run(module: ModuleType) -> list[tuple[Callable, Exception | None]]: ...
```

CLI:

```
python -m assertions <dotted.module>
```

## Behavior

### `run(module)`

For each function `f` returned by `discover(module)`, in order:

1. Call `f()` with no arguments.
2. If `f` returns normally, append `(f, None)` to the result list.
3. If `f` raises an instance of `Exception`, catch it and append
   `(f, exc)` to the result list.
4. If `f` raises a `BaseException` that is not an `Exception` (e.g.
   `KeyboardInterrupt`, `SystemExit`), do not catch it — it propagates
   out of `run` and aborts the rest of the run.

Returns the result list in `discover` order. No printing, no logging,
no side effects on the module.

### `python -m assertions <dotted.module>`

1. Parse `sys.argv`. Exactly one positional argument is required.
   Anything else is a usage error: print
   `usage: python -m assertions <dotted.module>` to stderr and exit 2.
2. Call `importlib.import_module(name)`. If this raises, the exception
   propagates and Python prints a traceback to stderr — the CLI does
   not wrap or pretty-print import errors. Exit code follows Python's
   default for unhandled exceptions (1).
3. Call `run(module)` to get the results.
4. For each `(func, exc)` in results, print one line to stdout:
   - `<func.__name__> ... ok` if `exc is None`
   - `<func.__name__> ... FAIL: <type-name>: <exc>` otherwise, where
     `<type-name>` is `type(exc).__name__` and `<exc>` is `str(exc)`.
5. Exit with code `0` if every result has `exc is None`, otherwise `1`.

Output goes to stdout. The per-test lines and the exit code are the
entire contract for this slice.

## Module layout

| File | Action | Purpose |
|------|--------|---------|
| `src/assertions/_runner.py`   | create | Implements `run(module)` |
| `src/assertions/__main__.py`  | create | CLI wrapping `run` |
| `src/assertions/__init__.py`  | modify | Re-export `run` |

`_runner.py` is a separate module because it has a distinct
responsibility (invoking discovered tests) from `_discover.py` (finding
them) and `_markers.py` (tagging them). Each file maps to one verb.

## Tests

### `tests/test_runner.py`

Direct tests for `run`:

1. Module with one passing test returns `[(f, None)]`.
2. Module with one failing `assert` returns `[(f, exc)]` where `exc` is
   an `AssertionError`.
3. Module with both passing and failing tests returns results in
   `discover` order with correct exception types per slot.
4. Module with no tests returns `[]`.
5. Test that raises a non-`AssertionError` `Exception` (e.g.
   `ValueError`) is caught and recorded.
6. Test that raises `KeyboardInterrupt` propagates out of `run` (verify
   with `assertRaises(KeyboardInterrupt)`).

### `tests/test_cli.py`

Black-box CLI tests using
`subprocess.run([sys.executable, '-m', 'assertions', ...])` so the
`__main__` entry is exercised end-to-end:

1. All-pass module: exit code 0; stdout contains `... ok` for each test.
2. Failing module: exit code 1; stdout contains `FAIL:` and the failing
   function's name.
3. Zero arguments and two arguments: exit code 2; usage line on stderr.
4. Unimportable module name: non-zero exit code; `ModuleNotFoundError`
   appears on stderr (the CLI does not catch it).

### Fixtures

Small modules under `tests/fixtures/runner/`:

- `all_pass.py` — two `@test` functions that succeed.
- `has_failure.py` — one passing, one failing via `assert`.
- `non_assertion_error.py` — one `@test` that raises `ValueError`.
- `keyboard_interrupt.py` — one `@test` that raises `KeyboardInterrupt`.
- `empty.py` — no `@test` functions.

`tests/fixtures/runner/__init__.py` is an empty package marker.

## Acceptance

- `from assertions import run` succeeds.
- `uv run python -m assertions tests.fixtures.runner.all_pass` exits 0
  and prints one `... ok` line per test.
- `uv run python -m assertions tests.fixtures.runner.has_failure` exits
  1, prints one `... ok` and one `FAIL: ...` line.
- `uv run python -m assertions` (no argument) exits 2 with the usage
  line on stderr.
- All tests under `tests/` pass; `uv run mypy` reports no issues.
