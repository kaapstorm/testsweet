# Design: `catch_exceptions` and `catch_warnings`

## Scope

Add two context managers used inside test bodies that **capture** —
they do not assert. Both yield a list; the user inspects the list with
plain `assert` statements.

- `catch_exceptions()` — captures `Exception`-subclass instances raised
  inside the block.
- `catch_warnings()` — captures warning instances emitted inside the
  block.

Earlier sketches called this `assert_raises` / `assert_warns`. The
new design (a) separates capture from assertion, leaving the latter to
the user's plain `assert` statements, and (b) yields a list rather
than asserting up-front. The README will gain a short example
demonstrating the new pattern.

Out of scope for this slice:

- Type filters (`catch_exceptions(of=ValueError)`) — capture-all is
  enough; the user filters via `assert`.
- Capturing `BaseException` (e.g. `KeyboardInterrupt`).
- Exposing warning metadata (filename, lineno, source) — only the
  `Warning` instance itself is exposed for now.
- Other capture helpers (e.g. `catch_logs`).

## Approach

Both context managers return a `list` from `__enter__`. The same list
object is mutated by `__exit__` (or, for warnings, by the underlying
`warnings.catch_warnings` machinery during the body), so the binding
made by `as` sees the populated list once the `with` block ends.

API symmetry between the two is intentional: identical shape, identical
idiom for the user.

## Public surface

```python
from assertions import catch_exceptions, catch_warnings


# catch_exceptions — list has 0 or 1 items.
with catch_exceptions() as excs:
    1 / 0
assert type(excs[0]) is ZeroDivisionError


# catch_warnings — list can have any number of items.
with catch_warnings() as warns:
    warnings.warn('Use new_func', DeprecationWarning)
assert len(warns) == 1
assert type(warns[0]) is DeprecationWarning
assert 'new_func' in str(warns[0])
```

## Behavior

### `catch_exceptions()`

Class-based context manager. No arguments.

- `__enter__` returns an empty `list[Exception]`.
- If the body raises an `Exception` (or subclass), `__exit__` appends
  the exception instance to the list and returns `True` (swallow).
- If the body raises a `BaseException` that is not an `Exception`
  (`KeyboardInterrupt`, `SystemExit`), `__exit__` returns `False`
  (propagate). The list stays empty.
- If the body completes normally, the list stays empty.
- The list contains 0 or 1 items by definition (a single raise
  terminates the body).

### `catch_warnings()`

Class-based context manager. No arguments.

- `__enter__` opens `warnings.catch_warnings(record=True,
  action='always')` internally and returns an empty `list[Warning]`.
- During the body, every emitted warning is captured by the inner
  `catch_warnings` machinery as a `WarningMessage` record.
- If the body raises an exception, the inner `catch_warnings` is
  closed and the exception propagates. The returned list is still
  populated with whatever warnings were captured before the raise.
- At `__exit__`, the inner `catch_warnings` is closed and the returned
  list is extended with `record.message` for each captured record (so
  the elements are `Warning` instances, not `WarningMessage` records).
- The list contains 0 or more items in emission order.

### Why `action='always'`

`warnings.catch_warnings(record=True, action='always')` ensures
warnings emitted in the block are captured regardless of the outer
warning filter state. Without `action='always'`, an outer filter set
to `'error'` would convert a warning into an exception (which would
escape the catcher into `catch_exceptions`-territory), and an outer
filter set to `'ignore'` would suppress emission entirely. Inside
`catch_warnings`, neither happens — every `warnings.warn(...)` results
in an entry in the list.

## Implementation sketch

```python
import warnings


class catch_exceptions:
    def __init__(self) -> None:
        self._excs: list[Exception] = []

    def __enter__(self) -> list[Exception]:
        return self._excs

    def __exit__(self, exc_type, exc, tb) -> bool:
        if exc is None:
            return False
        if isinstance(exc, Exception):
            self._excs.append(exc)
            return True
        return False  # propagate non-Exception BaseException


class catch_warnings:
    def __init__(self) -> None:
        self._warns: list[Warning] = []
        self._catcher = None
        self._records = None

    def __enter__(self) -> list[Warning]:
        self._catcher = warnings.catch_warnings(
            record=True, action='always',
        )
        self._records = self._catcher.__enter__()
        return self._warns

    def __exit__(self, exc_type, exc, tb) -> None:
        try:
            self._warns.extend(r.message for r in self._records)
        finally:
            self._catcher.__exit__(None, None, None)
        return None
```

Class-with-lowercase-name (`catch_exceptions`, `catch_warnings`)
mirrors the pattern of `contextlib.suppress` and `pytest.raises`.

## Tests

`tests/test_catches.py` (new), using `unittest`:

### `catch_exceptions`

1. Body raises an `Exception` → list has the instance; type matches.
2. Body raises a subclass of `Exception` → list has the instance.
3. Body raises a `BaseException` that isn't an `Exception`
   (`KeyboardInterrupt`) → propagates out of the `with` block; list is
   empty.
4. Body returns normally → list is empty.
5. Inside the `with` block (before any raise), `excs` is empty —
   confirms it isn't pre-populated.

### `catch_warnings`

6. Body emits one matching warning → list has the `Warning` instance;
   `type` matches; `str(warn)` contains the message text.
7. Body emits multiple warnings → list has all of them in emission
   order.
8. Body emits no warnings → list is empty.
9. Body raises after emitting a warning → exception propagates; list
   contains the warnings emitted before the raise.
10. Outer filter `simplefilter('error')`: a warning emitted inside
    `catch_warnings` is captured and does NOT raise (verifies
    `action='always'` overrides).
11. Outer filter `simplefilter('ignore')`: a warning emitted inside
    `catch_warnings` is captured (verifies `action='always'`).
